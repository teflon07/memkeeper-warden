## Notes for legal review (remove before finalizing)

This draft adapts the Apache ICLA v2.2. Points to confirm with counsel, given the
maintainer intends to keep the option of a commercial/hosted edition:

1. **Nonprofit clause removed.** The ASF preamble commits the Foundation to use
   contributions only consistent with its nonprofit status and bylaws. That is
   omitted here because the Project may pursue commercial offerings. Confirm the
   remaining grant (esp. "sublicense" in §2 plus "recipients of software
   distributed by the Project") supports a future relicense of the core to a
   different license, or add explicit relicensing language if stronger optionality
   is wanted.
2. **Contracting party.** "Project maintainer" is currently the individual
   operating the repository. If an LLC/entity is formed, name it here and align
   the LICENSE copyright holder.
3. **Patent grant (§3).** Confirm the termination-on-litigation scope is what you
   want.
4. **Corporate CLA.** §4 references a separate Corporate CLA; add a CCLA if you
   expect contributions from employees of companies.
